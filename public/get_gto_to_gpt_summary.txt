GTOtoGPT — Project Summary

GTOtoGPT (live at gtotogpt.com) is an AI poker-coaching app that answers No-Limit Hold'em strategy questions by combining Game Theory Optimal (GTO) solver data with OpenAI's GPT models. Rather than acting as a plain ChatGPT wrapper, it grounds its answers in a knowledge base built from PioSOLVER solver outputs plus a curated poker theory library, so it can explain not just what the optimal play is, but why — including exploitative and opponent-tendency reasoning that pure solvers can't offer. It's aimed at serious poker students, currently specialized for 100bb 6-max cash games, and runs on a $15/month subscription with one free question to start. The product is split across three repositories: a client (the web frontend), a server (the AI/RAG backend), and a converter (an offline data-ingestion pipeline).

The Client
The frontend is a Next.js 14 App Router application written in TypeScript 5.1 on React 18. It's styled with Tailwind CSS v3.3 and shadcn/ui atop Radix UI primitives, using the Montserrat font, Lucide icons, and a tri-color "GTO" brand palette. Users can tag questions with poker-specific parameters like pot type (single-raised vs 3-bet) and positional matchup (e.g. Button vs Big Blind). The chat experience streams answers token-by-token over a WebSocket (socket.io-client), with a live progress bar reflecting the backend's retrieval stages, while regular REST calls (Axios and fetch) handle conversation history and admin tasks. State is managed with Zustand and TanStack React Query, forms with react-hook-form and Zod, and authentication with Clerk — where the user's subscription tier and rolling usage limits are stored directly in Clerk metadata. It's deployed on Vercel with Vercel Analytics.

The Server
The backend is the brain of the product: a Node.js service built with Express 4 and Socket.IO 4 in TypeScript, using MongoDB (via the raw driver) for conversations and logs, and Upstash Vector as the RAG vector store. Its signature feature is a multi-stage LLM "agent" pipeline that processes each question through a chain of GPT-4o calls — first classifying whether the question is even poker-related, then deciding whether solver data would help, extracting the structured spot and board, embedding and querying matching solver outputs and theory lines from the vector database, and finally streaming a grounded GPT-4o answer back over the socket with retrieved data injected as context. Each stage reports a progress percentage to the client. Embeddings use OpenAI's text-embedding-3-small, auth and subscription tiers run through Clerk, billing through Stripe webhooks, validation through Zod, and rate limiting caps usage at 25 questions per six-hour window. Every LLM exchange is logged to MongoDB to build a fine-tuning dataset over time. As a long-running, stateful socket server, it runs on a traditional Node host rather than serverless.

The Converter
The converter is the offline ETL pipeline that builds the knowledge base — a developer-run TypeScript CLI (executed directly via ts-node) that isn't user-facing. Its core job is to drive the actual PioSOLVER application as a child process, loading binary .cfr solver save files organized by stack depth, position matchup, pot type, and board, then walking the solver's game tree to enumerate decision nodes. It hand-parses the solver's text and CSV output with regex, normalizes bet sizes and all-in thresholds, and shapes each result into natural-language-annotated text that an LLM can interpret. It also processes aggregate solver reports (equity, EV, and equity-realization stats per flop). Those cleaned outputs are then embedded with OpenAI's text-embedding-3-small and upserted into the Upstash Vector database (migrated from Pinecone) that the server queries at runtime. The converter additionally curates the OpenAI fine-tuning datasets — a poker-relevance classifier and question-to-query translators — that the server's pipeline relies on. Because it shells out to PioSOLVER's Windows executable, the core pipeline must run on Windows.

How It Fits Together
The three repos form a clear pipeline: PioSOLVER solves poker spots, the converter cleans and annotates that data and uploads embeddings (and trains helper models), the server retrieves the relevant solver data and theory for a given question and feeds it to GPT-4o, and the client streams the resulting answer to the user. It's a domain-specific RAG architecture purpose-built for poker.

In Short
A three-part AI poker coaching product: a Next.js + Tailwind + shadcn client that streams answers over WebSockets, an Express + Socket.IO + MongoDB server running a multi-stage GPT-4o RAG agent over an Upstash Vector knowledge base, and a TypeScript converter that drives PioSOLVER to turn raw GTO solver output into LLM-readable, embedded training data. Authenticated with Clerk, billed through Stripe, and grounded in real solver solutions, GTOtoGPT explains optimal poker strategy — and the reasoning behind it — far more accurately than a generic chatbot.
